# Claude 自然循环 vs /loop 对比

## Claude 自然循环

Claude 在**一次对话**中，按照 SKILL.md 的伪代码指令，连续多次调用工具完成任务。没有任何特殊机制，就是 Claude 理解了"重复执行"的意图后自发地一步步做下去。

### 执行示意

```
用户: /traffic-study 自动答题

Claude (一次对话内):
  ├─ 调用 Bash: capture → 第1题 JSON
  ├─ 分析答案 → "B"
  ├─ 调用 Bash: click --options "B"
  ├─ 输出: "第1题: xxx → 答案: B"
  ├─ 调用 Bash: capture → 第2题 JSON
  ├─ 分析答案 → "正确"
  ├─ 调用 Bash: click --options "正确"
  ├─ 输出: "第2题: xxx → 答案: 正确"
  ├─ ...
  └─ 输出: "20题全部完成!"
```

### 适用场景

- 连续完成一个有限任务（答完20题、处理一批文件）
- 步骤之间有依赖关系（上一题的结果影响下一步）
- 需要共享上下文（Claude 记住之前做了什么）

---

## /loop

Claude Code 的定时触发机制，类似 cron。每隔 N 分钟**启动一次新的独立执行**，每次执行之间互不关联。

### 执行示意

```
用户: /loop 5m /traffic-study check-status

Timer:
  ├─ [00:00] 触发 → Claude 独立执行一次 check-status → 输出结果
  ├─ [05:00] 触发 → Claude 独立执行一次 check-status → 输出结果
  ├─ [10:00] 触发 → Claude 独立执行一次 check-status → 输出结果
  └─ ... (直到手动取消或超时)
```

### 适用场景

- 周期性监控（每5分钟检查部署状态）
- 定时巡检（每小时检查服务健康度）
- 不需要上下文连续性的重复任务

---

## 对比总结

| 维度 | Claude 自然循环 | /loop |
|------|-----------------|-------|
| **触发方式** | 一次对话内连续执行 | 定时器，每隔 N 分钟触发一次 |
| **上下文** | 共享同一个对话上下文 | 每次是独立执行，无共享上下文 |
| **步骤间关系** | 有依赖，前一步的输出是下一步的输入 | 无依赖，每次执行独立 |
| **结束条件** | Claude 判断任务完成就停 | 手动取消或自动超时（7天） |
| **典型场景** | 连续答20题、批量处理文件 | 每5分钟检查一次状态 |
| **实现机制** | 无特殊机制，Claude 理解伪代码后自发重复 | CronCreate 工具创建定时任务 |
